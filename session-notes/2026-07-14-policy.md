# Session Policy — 2026-07-14

## TradingView Work Policy

- **DO:** Use `toolbox_call` → `tradingview` provider directly for all CDP/TradingView work
- **DO:** Acquire `tradingview_cdp_lock_acquire` before any chart tool call; release after
- **DO:** Use `toolbox_call` → `pine_guard` directly for Pine Script validation
- **DO NOT:** Dispatch to `tw-mcp` local Cascade coder for TradingView work — relay tools are the correct and preferred path
- **DO NOT:** Send any dispatch to any Cascade window without explicit per-session user approval

## Rationale
Relay tools (`toolbox_call`, `toolbox_list_tools`, `toolbox_status`, CDP lock tools) provide direct remote access to TradingView and Pine Guard providers. Dispatching to local coders adds unnecessary latency, requires human approval overhead, and bypasses the established relay architecture.

## Exception
Dispatch to `tw-mcp` only if a relay tool fails repeatedly AND the local coder is needed to diagnose provider behavior. Even then: explicit user approval required first.
